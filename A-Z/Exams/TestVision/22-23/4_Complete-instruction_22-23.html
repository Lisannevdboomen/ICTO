In this instruction, we explain the fundamental elements of TestVision.

The menus

This is the homepage. The following menus are important for creating the exam:



Folder structure

The instructor gains access to the root folder of a course, as in the example below: Research Methods. Within the menu 'questions' and 'tests', subfolders can be created that refer to a certain exams of the course. See the example below:

In most cases, the ICTO team has already created these subfolders for the instructor. But you can also create your own folders by doing the following:

  • Click on the settings icon in the upper right corner (see picture above).
  • A menu will open with which you can perform actions on your folder structure. Behind some actions there are letters in parentheses. These are keyboard shortcuts that correspond to the buttons on your keyboard.
  • You can click "add folder [m]" in the menu, or type "m" on your keyboard to create a folder.
  • Subfolders can be identified by three dashes under each arrow, as in the example above under "Archive 17/2018"

Menu setup

The 'questions' menu is shown below as an example:


Within the menu 'Questions' or 'Tests' you can view and change the folder structure on the left side.

In the lower right corner of the screen you can create a new item (a new question or a new test). The existing questions (or the test) are always shown on the right side of the screen.

Behind each question there are three symbols:

a|                     Change the name of the question (or the test)
i                       Information on the settings of the question (or test)
pencil            Editing a question (or a key)

If you want to select multiple questions (or tests) at the same time, use the boxes at the top right of the screen next to the settings icon.

Question status

When you're done making the questions, make sure you change the question status to "Approved." This way, you make it clear to TestVision that the questions are ready for the next phase. Select the questions and click on the settings icon in the upper right corner. Then click on: 'change question status'.

Tip: If you are unable to change the question status into Approved, it is likely that the questions have not yet been assigned criteria points. Click on the pencil on the right side of each question to do so!

Create a concept of the exam

Use or view this template to create a clear concept. The template helps to prepare teachers for a digital test. If a test is bilingual, this template can also help to identify bottlenecks.

The following considerations are important:
- Decide which question types you want to use
- Consider how many questions will be made in total

How to implement questions in TestVision

As a teacher, you fill in the questions in TestVision yourself. To enter questions, follow the next steps:

Log in and go to Questions

  1. Go to Testvision and login via 
  2. Click on the menu Questions
  3. Select the folder of the course on the left. If you don't have access to your course yet, please send an email to icto-csw@uva.nl
  4. Click on New in the lower right corner

Creating questions

  1. Type the text of the question in the input field beneath 'Stem'.
  2. Choose the type of question you want to create in the top right-hand corner of the screen.
  3. Enter the number of points for the question and some other question-specific properties such as random answer alternatives, partial scoring, and word counter.
    • Beware! If you have open questions, you will always have to fill in grading criteria at the bottom of the page. You can also use multiple criteria (a check-up model) for subsections that make up an essay question.
  4. Then click on the Save button in the lower right corner.
  5. Give the question a recognizable title so that you can always look up or reuse questions in later tests, use the following format:
    • [study] - [course] - [year] - [examination type] - [question]
      For example POL-IPRES-18-19-RESIT2-OPEN1A

Optional: import multiple choice questions from Word to TestVision

Keep our deadline in mind: at least 7 days before the start of the test, you can send the multiple-choice questions to us.

It is possible to import multiple choice questions from Word to TestVision. The ICTO team will import the Word document for you. Only with a minimum of 20 questions will the MC questions be imported for you. You can also add other types of questions, such as open questions, in TestVision yourself.

What should the document look like?

You can use this document: template MC questions ICTO

Each question must have a consistent format:

  • The correct answer is indicated by the sign: *
  • There must be no spaces before all questions and answers.
  • Special formatting such as font, underline, thickness or colour are not implemented (bold and italic are included)
  • Please do not use the enumeration function of Word.
    How do you remove this formatting? Copy the text and paste it back into the document with the function letter A (Text only).
  • Make sure that the answers to all questions are consistently uppercase or lowercase (and, if necessary, all end with a dot).
  • Only send the multiple-choice questionnaire, don't include any other elements such as an instruction text.

Send the Word document to icto-csw@uva.nl

The ICTO SW team will send an e-mail when the questions are ready in TestVision.

Go to the Questions menu to check the imported questions:

  • Always check if the import went well.
  • Make sure all question settings are correct.
  • Change the title of the question and use the following format:
    • [study] - [course] - [year] - [examination type] - [question]
      For example POL-IPRES-18-19-RESIT2-OPEN1A

Question order

You can offer questions in a fixed order or in a random order. You can also group questions and then decide by each group which of these two options you like to choose. Below is an explanation of what you can do with each option. 

  • Offer questions in a fixed order
    If you want to change the order of the questions, then you can click on the arrow keys beside each question on the right. A new small navigation menu will appear that allows you to move the questions up and down (you can then use the up/down arrows on your keyboard to scroll, this saves a lot of time and clicks). Make sure you have the option 'Random Question Order' on the top right of your screen, as it is on by default.
  • Offer questions randomly
    The option 'Random question order' is ticked in the top right of the screen by default. You can enable or disable the option by checking or disable the button at the top right of the screen.
  • Group questions and then choose by group
    You can use blocks: they allow you to group questions and then you can decide per group whether the sequence of questions should be on or off. Grouping questions into sub-blocks can be very useful if you want to group according to the type of question (e.g. one part multiple-choice questions, one part open questions), if you want to use a case or instruction text for a group of questions, or if you want to randomise some groups and not other blocks. Follow the instructions below to see how to create sub-blocks.

    • Block introduction text visible:
      Here you can place a text that should be visible when you enter the block. The text is available for each question within the sub-block. You can think of a general instruction text or a case study.

Properties: cut-off, test time, chance score, spell checker

Click properties in the upper right corner of your screen (at the Test menu).

The following window appears:



(1) Maximum test time
By default, the maximum duration of the key will be 60 minutes. Adjust this using the arrows or enter the number of minutes in the box.

(2) Cut-off
The cut-off is usually set to 55% (depending on the difficulty of the test). Because a cut-off is set, TestVision will automatically generate grades after the assessments have been recorded and thus convert the points/percentages to grades with one decimal place after the decimal point. The score will give the grade 1.0 when 0 points are obtained. This will be calculated up to grade 5.5, from grade 5.5 onwards the number of points/percentage will be equal to the grade again, so 5.5 is 55 points. Below 5.5, this recalculation will result in a slightly higher grade. Doubts about the cut-off? Ask the board of examination for advice.

(3) Grade scale
Grade scale allows tenths of a point for the final grade.

(4) Overview
In the overview, you can see very clearly when a pass (55%) will be obtained and how many points this will eventually have to be. This information can be shared with the student using the introduction text. There is no need to save, click the cross symbol (5) to close the window.

Grading in TestVision

Grading together with colleagues

Before you start assessing the exam, you have to agree on the division of grading between you and your fellow teachers. Who checks which question? Or who checks which students? You have the option of assessing by question (horizontal grading) or assessing by candidate (vertical grading).

All lecturers can look at each other's assessment and adapt or take over each other's work. So make sure to agree with each other on how the assessment will be done!

Where can I grade?

Grading in TestVision is very simple.

  1. First login to TestVision, use SurfConnext on the right side to do so.
  2. Click on the menu Grading
  3. Make sure that you are in the tab Grader
  4. Click on the button Grade on the right side of the test that you want to assess
  5. Two tabs are shown at the top. You have the possibility to assess per question (horizontal grading) or per candidate (vertical grading). Choose one of these options and make sure to check how your colleagues are grading the exam
  6. Now click on Grade next to the candidate or the question you want to assess (on the right)
  7. When grading, you will see the following elements from top to bottom:
    • The question
    • The answer given by the student
    • A grading box where you can fill in criteria
    • Feedback to a student (if preferred)
    • The assessment scheme
  8. Click on next to go to the next candidate
  9. Click on close when done

Analyze the quality of questions

Would you like to gain insight into the quality of your questions? TestVision automatically carries out an analysis of all questions.

Where can I find the analysis?

  1. Go to the Published tests menu
  2. Click on Results next to the test. By default, a 'ticket' version of the test is also ticked, do not change this.
  3. Click on Analysis in the lower left corner and select the Item Analysis tab.
  4. Below you will see an overview of the analysis per question
    (It is also possible to download this data by clicking on Download CSV in the lower right corner)

TestVision uses colour codes to indicate whether or not a question is of good quality according to their guidelines. Below you will find more information about the content of this quality analysis.

Explanation of the analysis values

In the columns you can find the following information from left to right:

  • Type: the type of question. If you don't know the abbreviation, then hoover over it to see what type of questions it is.
  • MaxSc: the maximum number of points that could be obtained for the question.
  • Chance: the chance score of the question.
  • D/P: whether it is a dichotomous (D) or polytomous (P) question. With a polytomous question, students can score a part of the points, with a dichotomous question it is only right or wrong.
  • #cpl: the number of times that the question is completed.
  • AvgSc: how many points the students have scored on average for the question. For multiple choice questions (and other dichotomous questions) this is equal to the p-value.
  • p/p': the degree of difficulty (p-value) of a question without partial scoring and without guess correction, meaning the proportion of students that have answered the question correctly. The value ranges from 0 to 1.  The higher the value, the easier the question was for the candidate group. 
  • p(corr): the degree of difficulty after the chance score is added.
  • rit: the distinctive character of the question. This involves looking at how students have made the question compared to how they have made the examination. This involves comparing how the students (who scored the highest on the examination as a whole) and the students who scored the lowest on the question created the question. The more students with a high examination score also score high on this question, the higher the rit-value. 
  • rir: this is a variation on the rit-value. With the rit-value, the score on the question is compared to the entire examination including the question itself, and with the rir value, this is compared to the examination excluding the question itself.

At the beginning of each row you will also see an i, if you click on it you can find more information about the question and possibly view a preview of the question.

Question quality indication by TestVision

Please note that the colour codes for each question only give an indication of the quality. A red question is not necessarily a bad question, but it may be wise to take a closer look at the analysis values for such questions.

The indication via colour coding of the question quality takes into account:

  • p- en p'-value;
  • a-values;
  • rit-values (only for dichotomous questions in not at randomly composed tests).

If an item is green, it means that these values meet the most important quality standards. Orange items have one or more values that deviate slightly to seriously from the quality standards. In the case of a red item, there is something wrong with various values. You can use this coding to quickly see which items you need to investigate in any case. The assignment of a colour is done as follows:

Green = 8 or 10 points
Orange = 6 points
Red = 0, 2 or 4 points 

Each item starts with a score of 10 points

  • 2 points deduction if the p- or p'-value < 0.40

  • 4 points deduction if the p or p value < 0.30 or > 0.90

  • 6 points deduction if the p or p value > 0.95

  • 2 points deduction if a diverter has an a-value greater than the a-value of a key

  • 2 points deduction as an a-value < 0.05 (up to 4 points deduction for this part)

  • 2 points deduction if the ride value < 0.30 and >= 0.20

  • 4 points deduction if the ride value < 0.20 and >= 0.10

Adjusting the results of questions (on an individual basis or for the entire exam)

Important: you can only change the results once the entire assessment (grading) has been validated. Go back to the main menu to see how to do this (Grading - step 2 - record your assessment).

Adjusting the result of a question for everyone

  1. Go to Published tests
  2. Click on Results next to the exam. Thereafter, you see that the exam is selected as well as a 'ticket' version of the exam, please do not change this selection!
  3. Click on Modify below
  4. Click on the Item Result tab above
  5. Here you can choose per question whether you want it to be considered correct for everyone (despite given answers), or if you would like to exclude the question from the entire exam. You can change the answer key of a multiple-choice question by clicking on the pencil on the right and then selecting another answer option. It is also possible to have both the original and the new answer key calculated correctly by checking the box 'Highest result'.
  6. Once you have made all the changes click on Check modifications. Here you can see how the score of the students changes as a result of the modifications.
  7. If you are sure you want to implement the changes, click on Apply modifications.
Adjusting the results for an individual student

  1. Go to the menu Sessions
  2. Click on Results next to the exam
  3. Find the student's last name in the list and select the student
  4. On the right-hand side of the 'Candidate details' window, click Modify
  5. A new window appears. Here you can choose per question whether you want it to be considered correct for the individual student, or if you would like to exclude the question for the student to the overall score. You can change the score (for open questions) by clicking on the pencil on the right.
  6. Click on Check modifications and check how the modifications alters the student's score. Click ok to approve the modifications.

Answer model

For inspection, it is important to ask yourself whether an answer model is needed for students. If, for example, you have created an exam with a number of open questions, it is useful for students to see during the inspection how the assessment is done. 

You are completely free to decide what the answer model may look like. The ICTO team asks you to do two things:

  • Please send the answer model in PDF
  • You can send the answer model up to 3 days in advance by e-mail. In that case, we will send the answer model to Topdesk, so that the students can open the answer model on their computer or laptop during the test. If it is not convenient to deliver the answer model before this deadline, you can still take it with you on USB or print it out for inspection!
