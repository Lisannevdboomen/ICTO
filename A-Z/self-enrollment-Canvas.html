About the self-enrollment system

It is possible to have the registration for lectures take place automatically in Canvas by enabling self-enrollment. This instruction focuses on working with a system in Canvas called "Group sets" (or commonly referred to as "Groups") and can be found in the Canvas course menu 'People'. Although there are many variations to implement, it all boils down to using 2 different methods which can be alternated:


  1. First come, first served   (View example)

    Students click JOIN or LEAVE to register for lectures. The 'First come, first served' method increases the chances of maximum occupancy because (as long as the maximum group size is not yet succeeded) students can register for any lecture provided in the list.

  2. Choose between a number of lectures each time   (View example)

    In this example, students can choose to join one lecture a week (click JOIN, SWITCH, or LEAVE). By using this method we are dividing the opportunity for students to register.

  3. Combined version

    A combined version could be something like this:

    Week 1: Students can choose between the lecture on Monday or Wednesday

    Week 2: Students can choose between the lecture on Monday or Wednesday

    Thereafter: Students are free to signup for any lectures (first come, first served)

Registration is always open and accessible to students. A maximum of 75 places will be set and if this number is exceeded, students can’t register for the particular lecture anymore. However, students can “JOIN” or “LEAVE” lectures, so the list can change until the last moment. If students register for a lecture, we expect them to be there. In case of changes, students need to remove their registration (click LEAVE) as soon as possible to provide a free spot for fellow students.

 

The 'First come, first served' method increases the chances of maximum occupancy because (as long as the maximum group size of 75 is not yet succeeded) students can register for any lecture provided in the list.

Instructions

1. Go to People in your course menu and select + Group set
 



2. Fill in the right details according to the picture below and click Save
 



3. Notice that you are now navigated to your new Group Set called "Lecture 1"
 

This Group Set is already pre-filled with a group (called "Lecture 1 1", by default). If you can't see any groups, click on + Group in the upper right to create a new group.



4. Navigate to the new group, click on the icon with three dots on the right and select Edit
 



5. Edit the name of this group (with details about the lecture) and click Save
 



6. The first lecture is done!
 

Please note: students can only self-enroll for one group within each Group Set, it is never possible to be part of two different groups within the same Group Set. With that in mind, we are going to create a new Group Set in order for the "first come, first served" method to work. This way, students can enroll themselves to all groups (i.e. lectures).



7. Click on the icon with three dots on the top right and select Clone group set
 



8. Add a new name to this group set and click Submit
 



9. A new Group Set appears (called 'Lecture 2').
  Head over to this new Group Set and edit the details of the lecture (i.e. edit the name of the new group). Repeat the steps above (cloning group sets) as many times as needed to finish up all lectures.

    Do you want to offer students an even chance of registering for lectures? In that case, cluster a number of lectures together (from which students have to choose) so that there are more seats to be distributed each time. For example: students can only choose one of the two lectures per week.

    2. Fill in the right details according to the picture below and click Save

    4. Navigate to the new group, click on the icon with three dots on the right and select Edit
     



    5. Edit the name of this group (with details about the lecture) and click Save
      (repeat step 4 and 5 for the other lecture as well)



    6. The first week is done!
     

    Please note: students can only self-enroll for one group within each Group Set, it is never possible to be part of two different groups within the same Group Set. With that in mind, the reason we have placed two groups within the same Group Set is because we want to force students to choose between one of these two lectures.

    8. Add a new name to this group set and click Submit
     



    9. A new Group Set appears (called 'Week 2').

    For larger courses, it might be useful to offer registration for lectures for one group at a time. In that case, we need to make use of a rotating system. The method described below can be used for this, however, please note that the rotating system relies partly on a basis of trust and random-checking (because although we ask certain groups to register first, everyone has access to the registration system).

    The rotating system will be used in combination with the "first come, first served" method.

    Instructions

    1. Create a rotating system
     

    Divide the tutorial groups ('sections') of your course into larger groups of about 75-100 students. For example: a course has 9 tutorial groups in total, each tutorial group contains 26/27 students (more or less).

    • Group A = tutorial group 1 + 2 + 3 (total of 80 students)
    • Group B = tutorial group 4 + 5 + 6 (total of 80 students)
    • Group C = tutorial group 7 + 8 + 9 (total of 80 students)

    First, we want to offer Group A the oppurtunity to register for a lecture (at least 24 hours before the lecture), so ask students from Group A to register for lecture 1. Next, if we haven't exceeded the total maximum of 75 students yet, we offer everyone from group B and group C the oppurtunity to register (within the last 24 hours). For each lecture, keep this rotating system going.

    Follow the instructions below to create the registration system. At the bottom of this instruction you'll find a page in which you can fill in the rotating system details and share it with students.


    Please note that you do not necessarily have to implement the above details, follow the instructions below for creating the registration system and alter further details to your own liking!
